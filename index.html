Transcripción Reporte de incidentes de datos

¡Saludos!

Les doy la bienvenida al tema “Reporte de incidentes de datos”.

Algunos puntos importantes dentro de este tema son:

  1. Preparación: Es la etapa en donde se comprueba que existan los recursos y herramientas necesarios para abarcar un incidente. Debe tomar en consideración:

    • Recursos de comunicación.

    • Hardware y software.

    • Recursos de análisis.

    • Recursos de mitigación y remediación.

  2. Detección, evaluación y análisis: Dentro de este tema, hay ciertas áreas por abarcar, que son las siguientes:

    • Detección, Identificación y Gestión de Elementos Indicadores de un Incidente.

    • Análisis.

    • Evaluación.

    • Clasificación de incidentes de seguridad de la información.

    • Priorización de los incidentes y tiempos de respuesta.

    • Declaración y notificación de incidentes.

  3. Contención erradicación y recuperación: Es necesaria una estrategia para contener el incidente y reducir los costos, daños y pérdida de información.

  4. Actividades Post-Incidente: Se incluye el reporte adecuado de los incidentes, las lecciones aprendidas, la implementación de medidas tecnológicas, disciplinarias y punitivas si es necesario, así como el registro en la base de conocimiento para alimentar los indicadores. Mantener un proceso de "lecciones aprendidas" después de un incidente grave, es útil para mejorar las medidas de seguridad y el proceso de gestión de incidentes.
